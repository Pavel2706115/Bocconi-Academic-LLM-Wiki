# Naming Conventions

## Folders

- Course source folders retain their original names under `Raw/Sources/` (e.g., `30001 STATISTICS`).
- Wiki subfolders use singular PascalCase: `Topics`, `Concepts`, `Entities`, `Projects`, `Logs`.

## Files

### Source Notes
- Retain original filenames from lectures/slides.
- New source notes: descriptive name matching the source title.
- Example: `Lecture 5 Slides with NOTES.md`, `Corporate-Finance-Chapter-3.md`

### Compiled Wiki Notes
- Use **kebab-case** with capital first letters: `Time-Value-of-Money.md`
- No spaces in filenames.
- Name should match the primary concept, topic, entity, or project.
- Examples:
  - `Wiki/Concepts/Net-Present-Value.md`
  - `Wiki/Topics/Corporate-Finance.md`
  - `Wiki/Entities/Bocconi-University.md`

### Index Files
- Always named `index.md` within their respective folder.
- Generated by `wiki_tool.py build`.

## Frontmatter Keys
- Lowercase with underscores for compiled notes: `source_count`, `created`, `updated`.
- PascalCase for source notes: `Title`, `Author`, `Reference`, `ContentType`, `Created`, `Processed`.

## Tags
- Always lowercase, singular: `concept`, `topic`, `entity`, `project`, `log`, `source`.
